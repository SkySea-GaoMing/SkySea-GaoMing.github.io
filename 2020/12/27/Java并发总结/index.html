面试题总结

什么是线程？

线程是操作系统能够进行运算调度的最小单位，它被包含在进程之中，是进
程中的实际运作单位

线程和进程有什么区别？

线程是进程的子集，一个进程可以有很多线程，每条线程并行执行不同的任
务。不同的进程使用不同的内存空间，而所有的线程共享一片相同的内存空
间，每个线程都拥有单独的栈内存用来存储本地数据。进程是系统进行资源
分配的调度的一个独立单位，进程是在一个正在系统中活动的指令集合，
具有生命周期和各种不同状态，也就是对正在运行程序的一种抽象

同步和异步的区别？

同步和异步通常用来形容一次方法调用。同步方法一旦开始，调用者必须等
到方法调用返回后才能继续后面的操作。异步方法更像一个消息传递，一旦
开始就会立即返回，调用者就会执行后面的操作

并行和并发？

严格意义上并行是多个任务同时执行，并发是多个任务交替执行

Java中活锁和饥饿和活锁有什么区别？

死锁就是多个线程相互竞争资源而造成的彼此阻塞，饥饿是指一个或多个线
程因为种种原因无法获得所需资源，活锁就是获得资源但还未加上锁时就又
将资源释放

如何在 Java 中实现线程？

有两种方式创建一个新的线程，一种方式是声明一个子类继承Thread，这个
子类需要重写run方法，一个子类的实例可以被分配空间并执行start方法。
另一种方式是声明一个类实现Runnable接口，这个类中实现run方法。创建
这个类的实例并作为参数传递给Thread类然后Thread类执行start方法

用Runnable还是Thread？

Java 不支持类的多重继承，但允许你调用多个接口。所以如果你要继承其他
类，当然是调用 Runnable接口。注意一点，有些博客会说Runnable更容易
实现资源共享，能多个线程同时处理一个资源，事实上忽视Thread本身也
可以当成一个Runnable对象作为参数传给Thread，只是一般不这样写

线程的生命周期是怎样的？

NEW新建，RUNNABLE 是可运行状态，WAITING 是等待状态，比如sleep和
wait，BLOCKED等待获取锁资源，TEMINATED是run方法执行完或发生异常

Java中interrupted 和 isInterrupted方法的区别？

前者会将中断状态清除而后者不会。 Java 多线程的中断机制是用内部标识
来实现的，调用实例方法interrupt()中断一个目标线程就会设置中断标识
为true，注意这里是目标线程而不是当前线程，目标线程是调用该方法的线
程对象所对应的线程，这个线程有可能并不是当前线程，比如在线程A 中调
用线程B的方法，这个时候线程A就是当前线程，线程B就是目标线程。调用静
态方法interrupted()来检查当前线程中断状态时，中断状态会被清零。非
静态方法isInterrupted()用来查询目标线程中断状态并且不会改变中断
状态标识。简单的说就是任何抛出InterruptedException异常的方法都
会将中断状态清零。无论如何，一个线程的中断状态有可能被其它线程调
用中断来改变，比如调用sleep wait join方法时如果中断了就会抛出
异常并清除中断

为什么wait和notify方法要在同步块中调用？

这两个方法存在竞态条件，必须在满足条件的情况下wait，如果不加锁的话很
可能在错误条件下wait，那么可能永远不会notify

Java多线程中调用wait() 和 sleep()方法有什么不同？

都可以让线程等待若干时间，wait可以被唤醒以及释放目标对象的锁，sleep
不会释放任何资源

join和yield的区别

join是阻塞当前线程，直到目标线程执行完毕。yield是一个静态方法，一旦
执行就会让出CPU，但是还可以进行CPU资源的争夺，注意yield不会释放锁
资源

Java中的volatile 变量是什么？

volatile是JAVA虚拟机提供的最轻量级的同步机制，它将具有两项特性：第一项
是保证此变量对所有线程的可见性，第二个特点就是禁止指令重排序。核心问题在
于能够保证使用之前是最新的数据，但是使用过程中其他线程可能修改这个值。禁
止指令重排序的关键是多指令了一个lock指令，这个lock指令相当于一个内存屏
障，指重排序的时候不能把后面的指令重排序到内存屏障之前的位置，lock不能
配合nop使用，将本处理器的缓存写入内存并且无效化别的处理器的缓存，写入
内存就已经意味着所有操作都完成。不能替代锁，性能要高于锁，主要的消耗就
是要在指令中插入内存屏障。并且JAVA内存模型对valatile的规则还有read
load use必须在一起 assign stoere和write必须在一起

如何避免死锁？

避免死锁最简单的方式就是阻止循环等待条件，将系统中所有的资源设置标志位、
排序，规定所有进程申请资源必须以一定的顺序（升序或降序）做操作避免死锁

Thread 类中的start() 和 run() 方法有什么区别？

start() 方法被用来启动新创建的线程，而且start()内部 调用了run()方法，这
和直接调用run()方法的效果不一样。当你调用run()方法的时候，只会是在原来的
线程中调用，没有新的线程启动，start()方法才会启动新线程

Java中Runnable和Callable有什么不同？

Runnable和Callable都代表那些要在不同的线程中执行的任务。它们的主要区别是
Callable的call() 方法可以返回值和抛出异常，而Runnable的run()方法没有这
些功能。Callable可以返回装载有计算结果的Future对象

为什么wait, notify 和 notifyAll这些方法不在thread类里面？

JAVA提供的锁是对象级的而不是线程级的，每个对象都有锁，通过线程获得。如果线
程需要等待某些锁那么调用对象中的wait()方法就有意义了。如果wait() 方法定义
在Thread类中，线程正在等待的是哪个锁就不明显了。简单的说由于wait，notify
和notifyAll都是锁级别的操作，所以把他们定义在Object类中因为锁属于对象

什么是ThreadLocal变量？

使用场合主要解决多线程中数据数据因并发产生不一致问题。ThreadLocal为每个线
程的中并发访问的数据提供一个副本，通过访问副本来运行业务，这样的结果是耗
费了内存，减少了线程同步所带来性能消耗，也减少了线程并发控制的复杂度

Java中活锁和死锁有什么区别？

不同之处在于处于活锁的线程或进程的状态是不断改变的，活锁可以认为是一种特殊
的饥饿，活锁和死锁的主要区别是前者 进程的状态可以改变但是却不能继续执行

怎么检测一个线程是否拥有锁？

在java.lang.Thread中有一个方法叫holdsLock()，它返回true当且仅当当前线程
拥有某个具体对象的锁
