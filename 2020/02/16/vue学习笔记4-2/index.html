计算属性的缓存以及与methods的对比

通过methods形式语法与计算属性基本相同，但是计算属性性能更高
如果多次打印fullname，methods每次都会重新执行函数，而计算属性有缓存机制不会重新执行除非数据改变

es6语法补充学习

let/const/var
let有块级作用域var没有块级作用域，for循环变量作用域
for(var i=0;i<length;i++)
{
click[i]:{
console.log(i);
}
}
这是一段简化代码，比如进行点击操作时，控制台永远都会打印length，如果改为let问题就会得到解决，可以理解为for的每一段循环都是一个块级
const: 将变量变为常量，不可以再次赋值，但是对象的属性可以修改，优先使用const
对象字面量增强写法
const name=’fd’; 直接将name赋给对象obj
const obj={name};
函数增强写法
const obj={ run: function(){}}变为 const obj={run(){}}

事件监听

与用户进行交互时监听点击、拖拽、键盘事件等
v-on:
作用：绑定事件监听器 缩写：@
调用一个方法时可以加括号也可以不加如果这个函方法不需要参数则没有区别
如果是一个需要传参数的方法，加括号但不传值则是undefined值，如果不加括号默认传递浏览器生成的Event对象
如果一个方法需要event对象，也需要其它参数时 需要传入$event

v-on修饰符:
stop 点击按钮时根据冒泡也会打印bbb，如果不希望冒泡可以改为 click.stop
prevent 阻止默认事件
KeyAlias 监听键盘点击事件，如果只监听enter键可以改为keyup.enter
once click.once 只有第一次点击有反应之后就失效
