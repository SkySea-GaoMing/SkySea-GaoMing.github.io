vue学习笔记4-4

vue学习系列4

组件化开发

注意

父组件向子组件传递数据时，如果通过v-model双向绑定props改变数据时会警告，此时应该在子组件定义data数据，将此数据传给父组件不会警告
方法的参数默认都是字符串，如果传number需要转换

父组件访问子组件$children|$refs

$children是一个数组只能通过索引操作数组中的组件对象一般不使用

$refs默认是一个对象 在要使用的组件中加入ref指定名字 就可以通过$refs.名字获取组件

子组件访问父组件（一般不使用）

$parent，与父组件耦合度太高不适合复用
$root 根组件

slot

译为插槽,目的使原有设备具备更多扩展性
组件的插槽 使封装的组件更加具有扩展性，让使用者决定组件的内部的一些内容展示什么
抽取共性到组件中，预留不同的插槽
slot中可以有默认值，如果不向cpn中输入内容就用默认值,如果多个值同时放入一个slot中都是替换值

具名插槽

slot标签中加入name属性，在cpn中如果需要加入内容就需要slot=name来绑定对应的slot

编译作用域

父组件模板的所有东西都会在父级作用域中编译，子组件模板的所有东西都会在子级作域中编译
也就是子组件中的data只能在template中使用，父组件中的data会在父级范围使用

作用域插槽

父组件替换插槽的标签但是内容由子组件提供
使用一个名字data获取array数据，使用slot-scope=name获取插槽对象，然后就可以在父级中使用子级数据

模块化开发

CommonJS规范：导入 导出（了解）
es6规范：export/import 每一个模块都有自己的空间不能使用别的模块的数据
export基本使用，导出之后别的模块可以使用
import 可以使用别的模块的数据
在b.js文件中导出 m.js文件中导入




webpack

webpack是一个现代的javascript应用的静态模块打包工具

前端模块化

例如commonJS,es6
webpack依赖node环境，node环境依赖各种包，npm工具管理各种包
将各种资源模块进行打包合成一个或多个包（bundle）
webpack打包成一个文件夹部署到服务器

安装webpack

在terminal中输入以下指令，webpack改为3.6.0版本

src保存源码，然后打包放入dist(distribution)文件夹，webpack会自动处理模块之间的依赖生成最终的js文件
项目的默认入口main.js，打包后的文件可以跟之前的组件一样引用
打包指令 webpack 原js文件 打包成的js文件

注意

文件路径一定要写对有时候有一个.有时候有两个.

webpack配置

创建webpack.config.js文件，在文件中可以指定入口和出口，打包的时候就会按设置的出入口打包，直接写webpack指令即可

如果没有package.json文件创建package.json文件 执行 npm init指令 直接回车创建，name就叫meetwebpack
webpack与npm run build映射，在package.json文件中scripts中

loader

处理css、图片等文件，将css代码打包到dist文件中
安装loader输入指令 npm install –save-dev css-loader
npm install style-loader –save-dev
创建css文件，在b.js文件中导入css
