    JSON基础

    Key:Value，类似map,Key是字符串，Value是任意类型，通过Key可以得到Value。

    1. map集合转为json对象
    2. javabean变为json
    3. String变为json,String有格式要求
    4. 文件变为json,先将文件变为字符串，字符串变为json
    5. 生成json文件
    6. JsonArray
    7. map转为jsonarray以及jsaonarray转为map

    web基础

    • 静态网页
    • 动态网页
      是否随时间、地点、用户操作的改变而改变

    架构

    1. CS
      每台计算机都有安装相应的客户端软件，如果软件升级，所有软件都要升级，需要维护每一台客户端
      软件，软件比网页版美观，响应服务器比较快
    2. BS
      服务端自己升级客户端不需要升级操作，只需要维护服务器，客户端通过浏览器直接访问网页

    常见状态码

    1. 404
      访问的资源不存在
    2. 200
      一切正常
    3. 403
      权限不足，文件设置不可见
    4. 300/301
      页面重定向
    5. 500
      服务器内部错误

    MVC设计模式

    • Model 模型
    • View 视图
    • Controller 控制器
    1. View
      负责页面的显示，与用户交互。html/css/js/vue
    2. Controller
      分发器，分发模型，接收各种请求，将请求跳转到模型处理，将处理的结果返回给请求处
      一般使用servlet
    3. Model
      各种功能，例如增删查改,使用Javabean实现，处理业务逻辑和处理数据的javabean

    Servlet

    1. doGet()方法
      接收所有请求方式为get的请求
    2. doPost()方法
      接收所有请求方式为post的请求

    三层架构

    • 表示层
    • 业务逻辑层
    • 数据访问层
    1. 表示层
      与客户交互，与后端交互
    2. 业务逻辑层
      具有一定逻辑，可拆分
    3. 数据访问层
      增删改查，具有原子性
