Git原理及配置

  • 原本Linux系统使用BitKeeper，但是后来开始收费，所以Linux大神自己写了Git
  • 版本控制系统分为两大类，可以控制版本切换到任意版本
  1. 集中式版本控制 cvs svn，每一次版本都需要访问中央仓库
  2. 分布式版本控制 git，每个使用者都有本地版本控制系统，通过服务器在不同计算机之间推送
  3. 本地使用git，服务器使用github或者gitLab，gitLab一般在公司内部使用
  • git的优势在于
  1. 可以建立本地版本库，不需要联网。
  2. 可以重写提交说明。svn不可以
  3. 一般情况可以后悔，也就是还原到上一步。
  4. 可以创建多个分支，将一个项目变为都多个分支branch
  5. 全量，也就是每一个版本都包含全部文件，时刻保持数据的完整性
  6. 分布式的含义是我的代码可以放到多个服务器中管理
  • git的四种状态，暂存区和对象区合起来称为version
  1. 将一个文件交给git管理。git init将一个文件变为已管理状态
  2. modified 已修改。修改工作区的文件
  3. staged 已暂存。add到暂存区
  4. commited 已提交。放入对象区
  • 本地工作目录先放到暂存区，暂存区中的东西再放入版本库，防止版本更新过于频繁
  1. git add
    本地工作目录到暂存区
  2. git commit
    暂存区到版本库（对象区）
  3. git push
    将自己的版本库推送到远程仓库（github)
  4. git pull
    远程仓库内容放入版本库
  5. git checkout – file
    如果某个文件已提交并且又进行了修改可以放弃修改还原到已提交状态

在git bash中配置用户名和邮箱，–global是给计算机一次性设置，–system是给当前系统
用户一次性设置, –local给当前项目一次性设置，如果同时设置越精确优先级越高

  • git config –global user.name “SkySea-GaoMing”

  • .gitignore文件中配不需要提交的文件

git常见操作及egit配置

第一次发布项目

本地到远程

  • git add . 当前全部文件到暂存区
  • git rm –cached 暂存区回到工作区
  • git reset head 暂存区回到工作区
  • git commit -m “注释内容” 暂存区到对象区，只有文件修改之后提交才有效
  • git push -u origin master 第一次提交
  • git rm file 删除的文件会被放入暂存区，如果要彻底删除需要 commit 也就是提交删除命令
  • git reset Head file 使用git方式删除后悔了,执行后回到工作区，执行 checkout指令即可
  • rm file 删除文件，回到工作区。然后执行 add 和commit才彻底删除文件
  • git log 查看提交日志，发现一长串数字就是id生成器生成的，用于区分哪一次提交,github中
    会记录所有的提交信息
  • git mv filename1 filename2 重命名
  • git commit –amend -m “重写最新一次提交的注释”
