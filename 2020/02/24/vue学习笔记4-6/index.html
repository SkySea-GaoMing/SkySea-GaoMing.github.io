vuecli-脚手架

cli Command-Line Interface
可以快速搭建vue开发环境以及对应的webpack配置
安装node和NPM
NPM Node Package Manager
之前已经安装了脚手架2.9.6，参考4-1
直接创建新项目vuesecond
vue init webpack vuesecond 创建项目文件夹
? project name (vuesecond) 这里默认项目名时vuesecond，直接回车
? Project description (A Vue.js project) 默认描述信息，可以修改没有什么影响
? Author (SkySea-GaoMing 1849731045@qq.com) 读取gitconfig
Runtime + Compiler 先选这个，以后用下一个
? Install vue-router? 暂时选择n,以后再安装
Use ESLint to lint your code? 检测代码规范，如果不规范就会报错
set up unit tests? 单元测试，一般不用
Setup e2e tests with Nightwatch? 端到端测试，一般用不到
管理项目使用NPM,yarn是新推出的也可以使用

vuecli2的目录结构解析

chrome使用v8引擎直接将javascript编译成二进制文件
之前如果要执行javascript代码必须要有html文件并且在浏览器中执行，node使用v8引擎可以在终端运行

build和config都是配置文件
node_modules当前项目依赖的包
src中是要开发的代码
src/assets 放静态文件
static 放静态资源，会复制到dist文件夹中
.babelrc 将es6转为es5
.editconfig 对代码进行统一规范
.gitignore 不需要上传服务器的内容

compiler+runtime和runtime-only的区别

runtime-compiler : template->ast->render->vdom->UI
runtime-only : render->vdom->UI 性能更高，代码量更少

vue-router详解

路由就是通过互联的网络把信息从源地址传输到目的地址的活动

前端渲染和后端渲染

后端渲染：jsp php
后端路由：后端处理url和页面之间的映射关系
缺点：整个页面都是由后端开发，html和Java代码混杂在一起
随着Ajax出现，前后端分离
前端渲染：浏览器中显示的网页中的大部分内容都是由前端写的js代码在浏览器中
执行最终渲染出来的网页
前端路由：单页富应用，整个网页只有一个html页面，静态资源服务器中可能只有一个html文件
一个css文件，一个javascript文件，url和页面组件的映射关系是由前端路由管理
url发生改变时会抽取相应的组件，整个页面并没有刷新

history.replaceState不会保留之前访问的页面
