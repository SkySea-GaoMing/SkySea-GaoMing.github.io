异常

部分图片参考 https://blog.csdn.net/qq_29229567/article/details/80773970
异常机制就是当程序出现错误的时候的处理机制。异常和错误的区别就是异常
是可以被处理的，错误一般是无法处理的

异常的分类

异常可以分为运行时异常和编译异常

  • 运行时异常 可以通过编译不能通过运行，如NullPointerException、
    IndexOutOfBoundsException
  • 编译异常 从语法角度必须进行相应的强制处理异常否则不能通过编译，如
    IOException、SQLException

异常处理

Java的异常处理机制可以让程序具有极好的容错性，当程序出现问题时，系统
会自动生成一个异常对象来通知程序，从而实现将业务逻辑代码和错误处理代
码分离

捕获异常

将业务逻辑代码放在try块中，将错误处理逻辑放在catch块中处理

如果在try块中出现异常系统会自动生成一个异常，这个异常对象会提交给Java
运行环境，这个过程就叫做throw，也就是抛出异常。Java环境接收到异常后会
寻找能够处理异常的catch块，如果找到合适的catch块就会交给catch块来处
理，这个过程就叫做catch，也就是捕获异常。如果Java环境找不到捕获异常
的catch块，那么运行环境就会终止

异常类的继承体系

每个catch块都是专门用于处理该异常类及其子类的异常实例，所以首先会判断
异常实例是否属于catch块后的异常或其子类。接下来看一下RuntimeException
的常见子类
catch块应该遵循先处理小异常后处理大异常的规则，子类异常的catch块应该
放在父类异常的catch块前面

多异常捕获

Java7以前每个catch块只能捕获一种类型的异常，Java7之后一个catch块可以
捕获多种异常

访问异常信息

如果程序想要获取异常对象的相关信息，可以通过catch块的异常形参获得，接下来
看一下Throwable源码，其中定义了所有异常对象公有的方法

finally

有些情况下系统打开了一些物理资源，比如数据库连接、网络连接和磁盘文件，
这些物理资源都必须显示回收，注意垃圾回收机制回收的是JVM内存中的空间。
接下来分析一下在哪里回收资源，如果在try中回收资源，那么程序运行到一
半可能产生异常，那么之后的代码就不会执行。如果在catch中回收资源，可
能程序不会产生异常那么catch块也就不会执行。所以异常处理机制提供了
finally块来回收资源，finally块一定会被执行

异常处理语法结构中try块是必需的，可以没有catch块或finally块，但两者
必须出现其一，就算没有catch块finally块也一定会执行。如果在finally块
中使用return或throw语句会导致try和catch中的return和throw语句失效

当try或catch块遇到return或throw时，首先会判断是否有finally块，如果
有finally块那么先执行finally块中的内容然后跳转到return或throw，如
果finally块中有return或throw，那么就不会再跳转到try或catch块

异常处理的嵌套

在try块catch块或finally块中包含完整的异常处理流程的情形称为异常处理
的嵌套，一般来说不要超过两层嵌套

自动关闭资源的try

当使用finally关闭资源时代码比较臃肿，必须手动关闭并且编写异常处理机
制。Java7增强了try块的功能，允许在try块后加括号，括号内可以声明初始
化多个资源，这些资源就是需要显示关闭的资源，try 块在语句结束后自动
关闭这些资源。注意这些资源必须实现AutoCloseable或Closeable接口，
实现这两个接口就必须实现close方法

Java9再次增强try语句，不要求在try后的圆括号声明并创建资源，只需要
自动关闭的资源有final修饰

throws

使用throws抛出异常的思路是：如果当前方法不知道如何处理这个异常，那么
该异常应该由上一级调用者处理，如果main方法也不知道如何处理异常那么也
可以使用throws声明抛出异常，将异常交给JVM处理。JVM对异常的处理方式就
是打印异常的跟踪栈信息，并中止程序运行。throws只能在方法签名中使用，
可以声明抛出多个异常类

方法重写时抛出异常的限制

使用throws抛出异常时有一个限制，子类方法抛出的异常类型应该是父类异常
抛出的方法类型的子类后相同，抛出的异常数不能更多

throw

当程序出现错误时会自动抛出异常，此外程序也可以自行抛出异常，自行抛出
异常用throw完成，每次只能抛出一个异常实例，注意运行时异常可以显示捕
获也可以不理会，Checked异常必须显示捕获

自定义异常类

自定义异常类需要继承Exception，如果希望自定义Runtime异常可以继承
RuntimeException
