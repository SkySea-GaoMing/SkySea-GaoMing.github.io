划分目录结构

开发新项目时可以先划分目录结构，使结构更加清晰

  • asserts 放图片和css资源
  • components 放公共组件，可以分为common和contents
  • views 放大视图
  • router 路由相关
  • store 公共状态管理
  • network 网络相关
  • common 公共js文件

css文件引入

normalize文件规范属性，可以在github中下载，base文件中引入normalize文件。

在App.vue中引入base.css

vue.config和editorcconfig

在项目下创建vue.config.js文件配置别名，以后引入路径时可以简化，vue-cli4好像无效

在App组件中

.editconfig对项目代码做一个统一的风格,在项目下创建文本文件

tabbar引入和项目模块划分

在components/common中创建tabbar文件夹创建TarBar.vue和TarBarItem.vue文件,可以复用
在components/contents中创建mainTabbar文件夹，放于本项目内容相关的内容，不能复用
