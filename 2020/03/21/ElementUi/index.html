ElementUI的使用

安装命令 npm install –save element-ui
在main.js中导入
import ElementUI from ‘element-ui’
Vue.use(ElementUI)

效果与相关代码

标签

  1. el-container 构建整个页面的框架，相当于一个div
  2. el-aside 构建左侧菜单，相当于左侧的div
  3. el-menu 构建左侧菜单内容，常用属性如下
  • default-openeds 默认展开的菜单，通过菜单的index值来关联，默认1,3打开
  • default-active 默认选中的菜单，通过菜单的index，相当于默认点击状态
  1. el-submenu 可展开的菜单栏，相当于el-menu的子节点
  • index 菜单栏的下标，必须是文本而不是数值
  • template 对应el-submenu的菜单名
  • i 设置菜单图标，通过class属性来设置
  • el-menu-item-group 设置分组，一般不用
  • el-menu-item 菜单的子节点，不可展开

VBlog

接下来分析一下VBlog中出现的样式

Form表单

  1. el-form里可以放置el-form-item，每一个表单域由一个Form-Item组件构
    成，表单域中可以放置各种类型的表单控件，包括 Input、Select、Checkbox、
    Radio、Switch、DatePicker、TimePicker
  2. 通过设置 label-position 属性可以改变表单域标签的位置，可选值为
    top、left，当设为 top 时标签会置于表单域的顶部

NavMenu

  1. 导航菜单可以分为两种，顶栏和侧栏。这里选用顶栏。导航菜单默认为垂
    直模式，通过 mode 属性可以使导航菜单变更为水平模式。另外，在菜单中
    通过submenu组件可以生成二级菜单
  2. 注意这里绑定了一个index属性，这里使用了vue-router模式，启动该
    模式会在激活导航时以index作为path进行路由跳转

LibraryIndex

注释掉的部分是接下来要编写的组件。这里使用Element提供的Container容器，
把整个页面分为侧边栏和主要区域两个部分。接下来配置这个页面的路由

  • el-container：外层容器。当子元素中包含 el-header 或 el-footer
    时，全部子元素会垂直上下排列，否则会水平左右排列
  • el-header：顶栏容器
  • el-aside：侧边栏容器
  • el-main：主要区域容器
  • el-footer：底栏容器

Books

需要注意的地方如下

  1. v-for指令 可以使用动态渲染
  2. el-tooltip 用于展示鼠标悬停时的提示信息
  3. slot 将标签中的内容插到父组件指定的地方，这里插入了el-tooltip中
    的content内容
  4. :src=”item.cover” 用于绑定把标签中的属性与data中的值绑定起来
  5. 分页使用 el-pagination

EditForm

  1. Dialog 弹出一个对话框
  2. title属性用于定义标题，它是可选的，默认值为空
  3. visible是否显示Dialog，支持.sync修饰符

ImgUpload

  1. action属性指定了上述操作对应的api，主要使用:before-remove、
    :on-success、:on-exceed
  2. multiple和:limit属性配合使用可以检测上传文件的数量
  3. 根据:on-success事件对应的方法，当该组件接收到后端返回的成功信息
    时，会触发父组件EditForm.vue的onUpload事件，把接收到的URL赋给图书
    信息表单的cover字段，这个URL 需要我们在后端根据资源存放位置生成。提
    交后数据库里就会保存服务器上的资源对应的URL
