IDEA介绍

IDEA是一款很爽的编辑器，旗舰版本通过学生邮箱可以免费使用

IDEA快捷键

  1. Ctrl+F 在当前文件进行文本查找，应该就是寻找关键字
  2. Ctrl+R 在当前文件进行文本替换，就是替换关键字
  3. Ctrl+Z 撤销
  4. Ctrl+Y 与撤销操作相反
  5. Ctrl+D 复制光标所在的行，并且插入到光标位置下面
  6. Ctrl+W 递进式选择代码块
  7. Ctrl+E 显示最近打开的文件记录列表
  8. Ctrl+N 根据输入的类名查找类文件
  9. Ctrl+G 在当前文件跳转到指定行处
  10. Ctrl+H 显示当前类的层次结构
  11. Ctrl+O 选择可重写的方法
  12. fori/sout/psvm +Tab 即可生成for循环/输出/main方法

Debug

Debug是IT从业者必会的一项程序调试技能，主要是以下三个方面

  • 追踪代码运行流程
  • 程序运行定位和调试
  • 线上问题追踪

IDEA中调试步骤

使程序进入Debug状态
与Run操作的结果一样，并未真正进入Debug状态

设置断点

设置断点才进入Debug状态，只需点击代码行左侧即可设置,然后重新进入Debug
状态，这时程序会在断点处暂停

8个按钮

接下来从左到右介绍8个基本按钮

  1. Show Execution Point (Alt + F10) 显示当前程序执行到哪一行
  2. Step Over (F8) 步过 单步向下执行而不进入方法，常用于不想进入方法
    体的情况
    执行步过操作此时没有进入方法中而是直接跳转到第4行
  3. Step Into (F7) 步入 如果当前行有方法就直接进入该方法，一般不会进入
    官方类库的方法中
  4. Force Step Into (Alt + Shift + F7) 强制步入 能够进入任何方法，一
    般用来查看底层源码
  5. Step Out (Shift + F8) 步出 从步入的方法体跳出到方法调用处，此时
    方法已经执行完毕但是还没赋值
  6. Drop Frame 返回到方法的调用处以便重新执行方法体
  7. Run to Cursor (Alt + F9) 直接运行到光标处，不需要打断点
  8. Evaluate Expression (Alt + F8) 计算表达式，可以查看变量也可以更改
    变量的值
    现在修改type属性为架构师

条件断点

给断点设置条件，只有满足条件时程序才会暂停

6个按钮

接下来从上往下介绍6个按钮

  1. Rerun (Ctrl + F5) 重新启动程序开始调试
  2. Resume Program (F9) 如果设置了多个断点，执行该操作会调到下一个断点
  3. Pause Program 没法点呀
  4. Stop (Ctrl + F2) 关闭程序
  5. View Breakpoints (Ctrl + Shift + F8) 管理所有断点
  6. Mute Breakpoints 断点哑音 所有断点失效，再点击一次又可以恢复，失效后
    按F9可以直接运行完剩下的程序，如果想要一个断点失效只需要取消enabled
