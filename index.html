总结{
新的属性：methods,该属性用于在Vue对象中定义方法
新的指令：@click，该指令用于监听某个元素的点击事件在指令点击
时发生
新的指令：@v-for，用于简化遍历操作
语法糖：v-on 简写
}


注意

data数据可以写在vue外 vue依然可以直接使用obj的数据

MVVM model view view model

view:视图层，前端开发中通常就是DOM层，主要作用是给用户展示各种信息
model:数据层，大部分来自服务器
viewmodel:视图模型层分为两部分
Data Binding 实现数据绑定，将model的改变实时反应到view中
DOM Listener DOM监听，当DOM发生一些事件（点击，滚动，touch），可以监听到并在需要的
情况下改变对应的Data

创建Vue实例传入的options

{
el:类型：string|HTMLElement 作用：决定Vue实例管理哪个DOM
data:类型：Object|Function（组件当中data必须是一个函数） 作用：Vue实例对应的数据对象
methods:类型：{[Key string]:Function} 作用：定义属于Vue的一些方法，可以在Vue中调用也可以在指令中使用
}

Vue的生命周期

生命周期：事物从诞生到消亡的过程
在github中下载稳定版本的vue源码
创建一个Vue实例时内部源码通过一系列初始化过程会回调创建的函数这时就可以知道内部进行到哪一步然后就会做

Mustache语法

在双括号中可以直接写变量或表达式

在标签中加入 v-once指令，其后不需要跟任何表达式（比如v-for后需要跟表达式），该指令表示元素和组件只渲染一次不会随数据的
改变而改变
v-html=”url” 会解析url中的字符串，如果不加v-html则只得到字符串，一般用于字符串带有标签需要解析的

v-text=”message” 跟Mustache作用相似，但是不灵活一般不使用
